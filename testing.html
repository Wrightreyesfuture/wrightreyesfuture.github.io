Testing Your Earth Science Website

This guide provides multiple methods to test your Earth Science tutor website outside of GitHub Pages.

Method 1: Local Testing (Simplest)

Testing Directly from Files

This is the simplest method but has some limitations with JavaScript functionality.

Extract the earth-science-tutor-final-fixed.zip file to a folder on your computer.

Open the extracted folder and double-click on index.html to open it in your web browser.

Navigate through the website to test basic functionality.

Note: Some JavaScript features might not work correctly when opening files directly due to browser security restrictions. If you encounter issues, try one of the other methods below.

Method 2: Local Web Server (Recommended)

Using Python's Built-in HTTP Server

This method creates a local web server that avoids most browser security restrictions.

Extract the earth-science-tutor-final-fixed.zip file to a folder on your computer.

Open a command prompt or terminal window.

Navigate to the extracted folder using the cd command.

cd path/to/earth-science-tutor

Start a Python HTTP server (Python must be installed on your computer):

# For Python 3 python -m http.server 8000 # For Python 2 python -m SimpleHTTPServer 8000

Open your web browser and go to: http://localhost:8000

To stop the server when you're done testing, go back to the command prompt and press Ctrl+C.

Using Node.js HTTP Server

An alternative if you have Node.js installed.

Install the http-server package globally (if not already installed):

npm install -g http-server

Navigate to the extracted folder in your command prompt or terminal:

cd path/to/earth-science-tutor

Start the HTTP server:

http-server -p 8000

Open your web browser and go to: http://localhost:8000

Method 3: Online Testing Services (No Installation Required)

Using Netlify Drop

This method allows you to quickly deploy your website to a temporary URL without any account setup.

Extract the earth-science-tutor-final-fixed.zip file to a folder on your computer.

Visit https://app.netlify.com/drop in your web browser.

Drag and drop the entire extracted folder onto the drop zone on the Netlify page.

Wait for the upload to complete. Netlify will automatically deploy your site and provide you with a temporary URL.

Use the provided URL to test your website. The site will remain available at this URL for a limited time.

Note: This is a great option for sharing the site with others for testing without having to deploy to GitHub Pages yet.

Using CodeSandbox

This method is useful for testing and making quick edits online.

Visit https://codesandbox.io/ and create a new static sandbox.

Extract the earth-science-tutor-final-fixed.zip file on your computer.

Upload the files to CodeSandbox by dragging them into the file explorer panel.

CodeSandbox will automatically serve your website and provide a preview.

Troubleshooting Tips

Check the browser console for errors: Press F12 in most browsers to open developer tools, then click on the "Console" tab to see any JavaScript errors.

Clear browser cache: Sometimes browsers cache old versions of files. Try clearing your cache (Ctrl+Shift+Delete in most browsers) or use incognito/private browsing mode.

Test in different browsers: If something doesn't work in one browser, try another (Chrome, Firefox, Edge, etc.).

Check file paths: Make sure all file paths in HTML, CSS, and JavaScript files are correct. Paths are case-sensitive on some servers.

Look for specific game errors: If games aren't loading, check the browser console for specific error messages related to the game initialization.

Deploying to GitHub Pages (Final Step)

Create a new repository named wrightreyesfuture.github.io on GitHub (if not already created).

Upload all files from the extracted folder to this repository.

GitHub will automatically deploy your site to https://wrightreyesfuture.github.io.

It may take a few minutes for changes to appear on the live site.
